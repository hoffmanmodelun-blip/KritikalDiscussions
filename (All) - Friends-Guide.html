Editing the website

A plain-English guide to updating the site while Justin is away — no coding, just clicking.

You cannot permanently break the site. Every change has an Undo button, and once a change is online, GitHub automatically keeps every earlier version — so the site can always be restored to how it looked before.

1

Get the current version of the site

Always start from the newest version. The editor now checks GitHub before publishing and stops safely if someone else updated the site while you were working.

  1. Double-click (ALL) - Kritikal-Editor - FINAL.html to open it in your browser.
  2. Click Open the latest live website.
  3. The editor loads the newest index.html directly from GitHub and remembers its exact version.
2

Choose a page and start editing

  1. Use the page dropdown at the top to jump anywhere on the site.
  2. Click Edit, then click the text, photo, button, card, or section you want to change.
  3. The bottom bar shows every safe action available for the selected item.

In Preview mode the site behaves like a normal visitor's would. In Edit mode, anything you click becomes changeable, and a bar at the bottom always tells you what's selected.

3

Make your changes

  • Text: click any heading or paragraph and type. Click elsewhere when done.
  • Photos: click a photo, then Replace photo in the bottom bar, and pick an image from your computer. Large photos are shrunk automatically.
  • Links & buttons: click one, then Edit link to change where it points (a Google Form, Instagram, an email, etc.).
  • Add or remove a person / instructor / card: click one, then use Add another (makes a matching copy you can edit), Remove, or the up/down arrows to reorder.
  • Insert a section: use Insert for text, photo, cards, statistics, gallery, FAQ, timeline, and call-to-action layouts.
  • Animations and design: select something and use Design for entrance effects, timing, alignment, text style, background, visibility, and page transitions.
  • Create a page: use Pages to add a blank page or a complete project page and place it in the website menus.
  • Move a whole section: select anything inside it, then use the section arrows, Duplicate section, Hide, or Remove section.
  • Made a mistake? Press the Undo arrow at the top. (Need a reminder of any step? Click the ? button in the editor.)
4

Publish so it goes live

Click the green Publish changes button. When your personal GitHub token is saved in Settings:

  1. Click Publish to the live site.
  2. The editor checks that your starting version is still current.
  3. If someone published first, the editor stops without overwriting them. Download a backup, reopen the latest website, and reapply your change.
  4. Wait about a minute, then open kritikaldiscussions.com to see it live.

Without a token, Download still creates the correctly named index.html for manual GitHub upload.

Three golden rules

  • One person edits at a time. Give the team a heads-up before you start, so two people don't publish over each other.
  • Always click Open the latest live website before editing. Don't reuse an old downloaded file from last week.
  • Keep changes small and check the live site after publishing. If it looks right, you're done.

If something looks wrong on the live site

Don't worry — nothing is lost. On the website's GitHub page, open the commit history (the clock/“History” link). Each entry is a saved version. Open a good earlier one and use Revert, or copy that version's index.html and re-upload it as in Step 4.

If you're unsure, it's completely fine to leave it and let Justin sort it out when he's back. The site keeps running either way.
