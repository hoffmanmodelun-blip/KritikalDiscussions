Pointing the domain to the website

A one-time, roughly 10-minute setup in Squarespace. After this you're done for good — you won't need to touch it again, and you're not involved in updating the site.

What this does: the site itself is hosted for free on GitHub. All you're doing is telling Squarespace that kritikaldiscussions.com should send visitors to GitHub's servers. You're only changing the records that point the website — email and everything else stays untouched.
1

The one value you'll need

You'll use this GitHub address in Step 4 — it's already filled in for you below:

jxia2027-cloud.github.io

That's where the website is hosted. The records you add simply send visitors there instead of to the old site.

2

Open the domain's DNS settings

  • Go to account.squarespace.com/domains and sign in.
  • Click kritikaldiscussions.com.
  • Open DNS (sometimes labelled DNS Settings).
If Squarespace says DNS can't be edited because the domain is “connected to a Squarespace site,” disconnect it from that site first (it should just be a registered domain, not attached to a Squarespace website).
3

Remove the records pointing to the OLD site

The domain currently sends visitors to the old Google Sites page, so those records have to go first — otherwise they override the new ones. In the DNS list, delete anything pointing the website elsewhere:

  • Delete any CNAME on host www whose value is ghs.googlehosted.com — that's the old Google Sites record.
  • Delete any existing A records on host @, plus any domain forwarding / redirect set on the root domain that sends it to the old site.
  • Keep the Squarespace Domain Connect record and any email / security records (MX or TXT, or anything labelled email) — those are unrelated to the website and shouldn't be touched.
If Squarespace shows a “forwarding” toggle that sends the domain to the old site, turn it off — a forward overrides the records you add below.
4

Add the new records

Click Add Record and create these five. The four A records all use host @; the CNAME uses host www.

TypeHostValue / Data
A@185.199.108.153
A@185.199.109.153
A@185.199.110.153
A@185.199.111.153
CNAMEwwwjxia2027-cloud.github.io
Leave TTL at the default (e.g. 4 hours) if asked. IPv6 “AAAA” records are optional and can be skipped — the four A records above are all that's needed.
5

Save, then tell Justin

Save your changes. That's everything on the Squarespace side.

Let Justin know it's done. On GitHub he'll press Check again, and once it verifies he'll switch on HTTPS. DNS changes usually take effect within an hour, but can take up to 24–48 hours to reach everyone.

If it's not working after a few hours

Still seeing a Squarespace parking page
A leftover default Squarespace A record or www CNAME is still there. Go back to DNS settings and remove it.
Want to check progress
Visit whatsmydns.net, enter kritikaldiscussions.com, choose A, and look for the four 185.199.x.153 addresses appearing around the world.
Site loads but shows a “not secure” warning
That clears once Justin ticks Enforce HTTPS on GitHub (it can take a day for that option to become available — normal).
Worried about email
As long as you kept the email/security records in Step 3, nothing about email changed.
